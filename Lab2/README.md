# Lab 2: Syntax Analysis Using CFG

This lab extends the lexical analyzer and adds syntax analysis using Bison/Yacc.

## Objective

The parser consumes the token stream generated by the lexer and validates whether the input program follows the grammar of the prescribed language.

The grammar supports:

- variable declarations
- assignment statements
- arithmetic expressions
- relational expressions
- boolean expressions
- `if-else` statements
- `while` loops
- block statements using `{ }`
- `print` statements

## Files

- `lex.l`: Flex lexer that returns tokens to the parser
- `par.y`: Bison grammar for syntax analysis
- `input.txt`: Official evaluation program used for parsing

Generated files such as `lex.yy.c`, `par.tab.c`, and `par.tab.h` are produced during compilation.

## Output

If parsing succeeds, the program prints:

- token stream
- parse tree
- leftmost derivation
- rightmost derivation

## How To Run

```bash
bison -d par.y
flex lex.l
gcc par.tab.c lex.yy.c -o parser
./parser
```

## Important Parser Features

- operator precedence and associativity are declared in `par.y`
- expressions are parsed according to grammar rules
- the parser builds a tree representation of the program structure
